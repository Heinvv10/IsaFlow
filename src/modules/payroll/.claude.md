# Module: Payroll

## Purpose
South African payroll processing — employee management, payroll runs, PAYE/UIF/SDL tax calculations, payslip PDF generation.

## Critical Rules
- ALWAYS scope queries with `company_id` (multi-tenant — fixed 2026-03-29)
- All API routes MUST use `withCompany` (not `withAuth`) — fixed in migration 249
- Tax calculations are legally binding — unit tests REQUIRED for any changes
- PAYE uses SA tax brackets, UIF is 1% employee + 1% employer, SDL is 1% of total payroll

## Key Files
| File | Purpose |
|---------|---------|
| `payrollService.ts` | Employee CRUD, payroll run processing, tax calculations |
| `payrollTypes.ts` | TypeScript interfaces |
| `payrollTaxTables.ts` | SA tax brackets, UIF rates, SDL thresholds |

## API Endpoints
| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET/POST | `/api/payroll/employees` | List/create employees |
| GET/PUT | `/api/payroll/employees-detail` | Single employee CRUD |
| GET/POST | `/api/payroll/payroll-runs` | List/create payroll runs |
| POST | `/api/payroll/payroll-runs-action` | Complete/reverse runs |
| GET | `/api/payroll/payroll-runs-detail` | Run details with line items |
| GET | `/api/payroll/payslip-pdf` | Generate payslip PDF |

## Database Tables
- `employees` — company_id, employee_number, name, ID number, tax number, bank details
- `payroll_runs` — company_id, period, status (draft/completed/reversed), totals
- `payroll_run_lines` — per-employee breakdown (gross, PAYE, UIF, SDL, net)
