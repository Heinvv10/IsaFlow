# Auth Module

Authentication and security services for ISAFlow — two-factor authentication, session management, and identity verification.

## Services (1)

| Service | Purpose |
|---------|---------|
| `twoFactorService.ts` | TOTP-based 2FA — setup, verification, backup codes, trusted devices |

## Key Functions

### twoFactorService.ts
- `generateTOTPSecret(userId, email)` — Create TOTP secret + QR code data URL
- `verifyTOTPSetup(userId, code)` — Verify initial setup, generate backup codes
- `verifyTOTPCode(userId, code)` — Verify during login (checks TOTP + backup codes)
- `disable2FA(userId, code)` — Disable with code verification
- `get2FAStatus(userId)` — Check enabled/method/verified status
- `addTrustedDevice(userId, fingerprint, name)` — Trust for 30 days
- `isTrustedDevice(userId, fingerprint)` — Check trusted status
- `getTrustedDevices(userId)` — List devices
- `removeTrustedDevice(userId, deviceId)` — Revoke trust

## API Endpoints

| Path | Methods | Auth |
|------|---------|------|
| `/api/auth/2fa-setup` | POST | withAuth |
| `/api/auth/2fa-verify-setup` | POST | withAuth |
| `/api/auth/2fa-verify` | POST | tempToken (no full auth) |
| `/api/auth/2fa-disable` | POST | withAuth |
| `/api/auth/2fa-status` | GET | withAuth |
| `/api/auth/2fa-trusted-devices` | GET, DELETE | withAuth |

## Dependencies

- `otpauth` — TOTP generation/verification
- `qrcode` — QR code data URL generation

## Key Rules

- Auth APIs use `withAuth` middleware (NOT `withCompany`)
- 2FA secrets encrypted at rest in `user_2fa` table
- Backup codes are single-use, hashed in database
- Login flow: password → check 2FA → if enabled, return tempToken → verify code → issue full JWT
- Owner/Admin roles get banner encouraging 2FA setup
- Trusted devices auto-expire after 30 days
