# Admin Module

Platform administration for ISAFlow SaaS operations — company management, user management, billing, feature flags, and system analytics.

## Services (12)

| Service | Purpose |
|---------|---------|
| `adminCompanyService.ts` | Company CRUD, suspension, metadata management |
| `adminUserService.ts` | Platform user management, role assignment |
| `adminDashboardService.ts` | System-wide metrics and KPIs |
| `analyticsService.ts` | Usage analytics, company activity tracking |
| `announcementService.ts` | System-wide announcements to all users |
| `auditService.ts` | Admin-level audit logging (separate from accounting audit) |
| `billingService.ts` | Subscription management, plan assignment |
| `dataExportService.ts` | Platform data export for compliance |
| `featureFlagService.ts` | Feature flag management, per-company feature gating |
| `impersonationService.ts` | Admin impersonation of company users |
| `invoiceService.ts` | Platform billing invoice generation |
| `planService.ts` | Subscription plan CRUD (Starter, Professional, Enterprise) |

## Middleware

| File | Purpose |
|------|---------|
| `withAdmin.ts` | Admin role guard — restricts to super_admin users |
| `withFeature.ts` | Feature flag gate — checks company has access to feature |

## Key Rules

- All admin APIs live under `pages/api/admin/`
- Admin middleware uses `withAdmin()` (NOT `withCompany()`)
- Admin pages at `pages/admin/`
- Separate from accounting module — different auth scope
- Feature flags stored in `platform_features` table
- Plans stored in `platform_plans` table
